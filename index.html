指数构建与方法说明

中国金融机构数字化能力指数

01

样本登记

按机构、年份、报告类型和原始路径生成清单，每份文件保留唯一 `result_id`。

02

文件映射

将 PDF 上传为远端 `file_id`，并保留 `file_id`、本地路径与机构元数据的映射关系。

03

领域评分

八个领域测度任务分别读取全文，在对应能力域内完成原文识别、档位判断和逐项评分。

04

原文依据约束

每次调用要求输出分项 0-5 整数分和评分理由；缺少明确原文依据时按 0 分处理。

05

结构化解析

脚本解析严格 JSON，并记录 `chat_id`、token、耗时、缺失项和原始响应。

06

合并与标准化

同一机构-年份存在多份公开报告时，按能力项保留更高披露依据对应的分数。

07

质量控制

执行分数范围、理由文本、路径映射、人工抽样、外部效标和关键词法对照检查。

分项测度

八个能力域的测度分工

每个测度任务对应一个能力域，评分指令包括任务边界、能力项定义、0-5 档位、原文依据要求和 JSON 输出格式。领域拆分有助于区分治理、技术、数据、服务与审慎管理等不同维度。

领域一

金融科技治理体系

9 项：组织协调、战略规划、数字文化、生态建设、科技伦理和主体责任。

领域二

数据要素潜能释放

8 项：数据规划、治理体系、数据架构、数据标准、数据质量、数据应用、共享和安全。

领域三

新型数字基础设施

13 项：数据中心、容灾、安全、运维、绿色能效、网络、IPv6、云和安全计算。

领域四

关键核心技术应用

7 项：产学研合作、供应链稳定、核心技术掌控、分布式架构、云化和价值创造。

领域五

数字化经营动能

12 项：敏捷创新、中台建设、自动化风险防控和数智化营销能力。

领域六

金融服务智慧再造

11 项：流程再造、线上线下渠道、小微、农村、供应链、绿色和无障碍服务。

领域七

金融科技审慎管理

11 项：监管科技、合规数据、风控能力、风险信息库、新技术和算法管理。

可持续化发展基础

4 项：金融科技人才选育、标准制定和标准实施。

评分指令

评分指令与输出格式

system: 仅依据报告原文和给定分级标准完成金融机构数字化能力评分。
system: fileid://<file_id>
user: <领域评分标准> + <严格 JSON 输出要求>

model: qwen-long
temperature: 0
output: {"维度1":{"指标1":0},"评分原因":"逐项说明依据、页码或段落位置"}

评分规则

评分约束

  • 每个指标必须给出 0 到 5 的整数分。
  • 较高档位默认要求较低档位全部满足。
  • 评分依据必须来自报告原文，尽量定位到页码、章节、段落或表格。
  • 与数字化关系弱或无明确披露的内容不纳入评分。
  • 不能根据机构规模、行业常识或泛化口号推断高档位。

评分结果示例

评分依据与档位判定

样例同步列示原文依据、档位判断、分项得分和未升档说明，用于展示具体分数的形成过程。

样例结构 数据要素潜能释放
  1. 原文依据

    报告披露战略规划、数据治理、线上贷款、风控系统、电子渠道和数据安全管理等具体实践。

  2. 档位判断

    逐项匹配“数据发展规划、数据治理、数据架构、数据标准、数据质量、数据应用、数据共享、数据安全”的档位要求。

  3. 分项得分

    示例输出为 4、3、3、3、3、4、3、3，均为 0-5 整数档。

  4. 未升档原因

    未披露实时数据平台、量化评价体系、行业级共享输出或 AI 驱动数据安全能力时，不给更高档位。

指数计算

能力域和总指数计算

能力域指数等于本域实际分数之和除以该域最高可能分数。总指数对 75 个能力项采用同一标准化逻辑。

Domain = Σ item score / (5 × item count)
Overall = Σ 75 item scores / (5 × 75)

公开版只发布机构-年份指数和八个能力域得分；内部过程表保留评分原因、文件路径、`result_id` 和 `file_id`。

质量控制与有效性检验

质量控制与外部验证

为降低披露强度差异和自动评分误差的影响，本项目结合结构化校验、人工抽样、金融科技专利、关键词法对照和单项重测识别潜在偏差，并明确指数的解释边界。

结构化校验

主表和文件索引均为 8,804 行；原因单元格 70,064 / 70,432 非空，覆盖率 99.48%。

人工抽样校验

覆盖高分、中分、低分，覆盖银行、证券、保险，以及年报和社会责任报告。

可疑表述核查

对“推断、未直接、接近”等高风险表述做二次标记，优先检查高分且含推断性理由的样本。

单项指标重测

针对 2-4 数据标准做全量重测，7,214 条记录最终全部解析完成，并保留两轮补跑和人工覆盖记录。

外部效标检验

金融科技专利样本匹配 41 家银行、384 个银行-年份观测；年度专利流量与指数均值相关系数为 0.855。

关键词法基准

传统关键词法作为独立基准，记录词典、OCR、低文本质量标记和频率指数生成过程。

适用边界

指数测量的是公开披露文本中的数字化能力信息，不把口号式表述等同于真实能力，也不把相关性结果表述为强因果结论。外部效标和替代方法用于增强有效性，而不是替代报告级评分依据。
