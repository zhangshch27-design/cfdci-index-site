01

报告登记

按机构、年份、报告类型和原始路径生成稳定清单，每份文件保留唯一 `result_id`。

02

文件账本

将 PDF 上传为远端 `file_id`，保留 `file_id`、本地路径和机构元数据的一一映射。

03

领域 Agent

八个 Agent 分别读取全文，在对应能力域内执行证据抽取、档位判断和逐项评分。

04

证据链输出

每次调用要求输出分项 0-5 整数分和“评分原因”，无明确证据一律给 0。

05

结构化解析

脚本解析严格 JSON，记录 `chat_id`、token、耗时、缺失项和原始响应。

06

合并与标准化

同一机构-年份存在多份公开报告时，按能力项保留更高披露证据对应的分数。

07

质检与验证

执行分数范围、原因文本、路径映射、人工抽样、外部效标和关键词法对照检查。

Eight domain agents

八个 Agent 如何工作

每个 Agent 只处理一个能力域，提示词包含角色、能力项定义、0-5 档位、证据约束和 JSON 输出格式。领域拆分避免把治理、技术、数据、服务和审慎管理混为一体。

Agent 1

金融科技治理体系

9 项：组织协调、战略规划、数字文化、生态建设、科技伦理和主体责任。

Agent 2

数据要素潜能释放

8 项：数据规划、治理体系、数据架构、数据标准、数据质量、数据应用、共享和安全。

Agent 3

新型数字基础设施

13 项：数据中心、容灾、安全、运维、绿色能效、网络、IPv6、云和安全计算。

Agent 4

关键核心技术应用

7 项：产学研合作、供应链稳定、核心技术掌控、分布式架构、云化和价值创造。

Agent 5

数字化经营动能

12 项：敏捷创新、中台建设、自动化风险防控和数智化营销能力。

Agent 6

金融服务智慧再造

11 项：流程再造、线上线下渠道、小微、农村、供应链、绿色和无障碍服务。

Agent 7

金融科技审慎管理

11 项：监管科技、合规数据、风控能力、风险信息库、新技术和算法管理。

Agent 8

可持续化发展基础

4 项：金融科技人才选育、标准制定和标准实施。

Prompt contract

统一提示词契约

system: 你是金融机构数字化能力测度助手，只依据报告原文和给定分级标准评分。
system: fileid://<file_id>
user: <领域评分标准> + <严格 JSON 输出要求>

model: qwen-long
temperature: 0
output: {"维度1":{"指标1":0},"评分原因":"逐项说明证据、页码或段落位置"}

Scoring discipline

评分约束

  • 每个指标必须给出 0 到 5 的整数分。
  • 较高档位默认要求较低档位全部满足。
  • 证据必须来自报告原文，尽量定位到页码、章节、段落或表格。
  • 与数字化关系弱或无明确披露的内容不纳入评分。
  • 不能根据机构规模、行业常识或泛化口号推断高档位。

Evidence-analysis-score chain

评分样例如何被复核

公开展示样例时，不只给出最终指数，而是同时呈现原文证据、Agent 判断、分项得分和未升档原因，让读者能看到“为什么是这个分数”。

样例结构 数据要素潜能释放
  1. 原文证据

    报告披露战略规划、数据治理、线上贷款、风控系统、电子渠道和数据安全管理等具体实践。

  2. Agent 分析

    逐项匹配“数据发展规划、数据治理、数据架构、数据标准、数据质量、数据应用、数据共享、数据安全”的档位要求。

  3. 分项得分

    示例输出为 4、3、3、3、3、4、3、3，均为 0-5 整数档。

  4. 未升档原因

    未披露实时数据平台、量化评价体系、行业级共享输出或 AI 驱动数据安全能力时，不给更高档位。

Index formula

能力域和总指数计算

能力域指数等于本域实际分数之和除以该域最高可能分数。总指数对 75 个能力项采用同一标准化逻辑。

Domain = Σ item score / (5 × item count)
Overall = Σ 75 item scores / (5 × 75)

公开版只发布机构-年份指数和八个能力域得分；内部审计表保留评分原因、文件路径、`result_id` 和 `file_id`。

Quality control and validation

质检与外部验证

审稿意见关注指数是否只是披露强度。本项目用结构化审计、人工抽样、金融科技专利、关键词法对照和单项重测来暴露误差来源，并限定结果解释边界。

结构化审计

主表和文件索引均为 8,804 行；原因单元格 70,064 / 70,432 非空，覆盖率 99.48%。

人工抽样

覆盖高分、中分、低分，覆盖银行、证券、保险，以及年报和社会责任报告。

风险词扫描

对“推断、未直接、接近”等高风险表述做二次标记，优先复核高分且含推断性理由的样本。

单项重测

针对 2-4 数据标准做全量重测，7,214 条记录最终全部解析完成，并保留两轮补跑和人工覆盖记录。

外部效标

金融科技专利样本匹配 41 家银行、384 个银行-年份观测；年度专利流量与指数均值相关系数为 0.855。

替代方法

传统关键词法作为独立基准，记录词典、OCR、低文本质量标记和频率指数生成过程。

解释边界

指数测量的是公开披露文本中可验证的数字化能力证据，不把口号式表述等同于真实能力，也不把相关性结果表述为强因果结论。外部效标和替代方法用于增强有效性，而不是替代报告级证据链。
